Temas

Banco financeiro

Tipos de usuário

  • Correntista
  • Caixa
  • Gerente

Funcionalidades

  • Abertura de conta corrente
  • Transferência de dinheiro
  • Pagamento de conta
  • Empréstimo com desconto em folha, a ser solicitado pelo correntista e aprovado ou reprovado pelo gerente
  • Fechamento de conta corrente, possível somente com a conta zerada e empréstimos quitados
  • Mecanismo para que outro site redirecione ao banco para fazer um pagamento com débito em conta


Site de venda de ingressos (ex: https://www.sympla.com.br/)

Tipos de usuários

  • Organizador (podendo ser uma pessoa física, um artista, uma banda, uma empresa etc.)
  • Comprador
  • Gerente de contas

Funcionalidades

  • (Organizador) Propor novo evento
  • (Gerente de contas) Aprovar ou reprovar evento
  • (Comprador) Comprar ingresso, passando pelo sistema de pagamentos e perguntando se usuário deseja adicionar evento à sua agenda
  • (Comprador) Entrar em fila de espera, sendo informado por e-mail se houver cancelamento
  • (Comprador) Transferir ingresso
  • (Comprador) Cancelar compra de ingresso, liberando para quem estiver em fila de espera
  • (Organizador) Cancelar evento, informando por e-mail os compradores e oferecendo opções de reembolso


Agenda de compromissos (ex: https://calendar.google.com/)

Tipos de usuários

  • Dono da agenda
  • Terceiro

Funcionalidades

  • Criar nova conta
  • Criar novo evento (dono)
  • Remover evento (dono)
  • Conceder permissão a terceiro para criar eventos
  • Criar novo evento (terceiro)
  • Cancelar evento (terceiro)
  • Configurar avisos por e-mail e/ou outras formas (SMS, WhatsApp etc.)
  • Web service para adicionar evento a uma agenda qualquer (que tenha autorizado isso previamente)


Ambiente virtual de aprendizagem (ex: https://www.cmp.ifsp.edu.br/moodle)

Tipos de usuários

  • Aluno
  • Professor
  • Administrador

Funcionalidades

  • (administrador) Criar conta para aluno ou professor
  • (administrador) Criar turma
  • (professor) Adicionar ou remover aluno a/de turma
  • (professor) Subir material em forma de URL ou arquivo
  • (professor) Criar tarefa, que irá receber arquivos dos alunos durante um período de tempo
  • (professor) Avaliar tarefa de aluno, com nota e texto
  • (professor) Enviar mensagem para todos os alunos
  • (todos) Recuperar senha


Xadrez (ex: https://www.chess.com/)

Tipos de usuários

  • Jogadores

Funcionalidades

  • Jogar contra computador (escolhendo nível de dificuldade)
  • Jogar contra outra pessoa conectada
  • Jogar com relógio (modo relâmpago) ou sem (podendo pensar à vontade)
  • Trocar tema (cores do tabuleiro e estilo das peças), salvando preferências na conta do usuário
  • Consultar partidas anteriores jogadas, podendo ver data e hora, lances da partida e resultado individual (vitória/empate/derrota), além do total de vitórias, empates e derrotas até o momento
  • (Opcional) Robô em JavaScript que joga como se fosse uma pessoa


Site de fazer sites (ex: https://sites.google.com/new)

Tipos de usuários

  • Criador
  • Visitante

Funcionalidades

  • Criar nova página, podendo inserir título, subtítulo, imagem etc.
  • Criar links entre páginas

Sistema de gestão de pagamentos (ex: https://moip.com.br)

Tipos de usuários

  • Pagador
  • Recebedor

Funcionalidades

  • (Pagador) Realizar um pagamento via cartão de crédito, boleto bancário
    • Para cartão de crédito, deve-se apenas validar de acordo com alguns critérios (número válido de acordo com algum algoritmo oficial, data de vencimento etc.). Não precisa enviar o pagamento à operadora de cartão de crédito. Caso dados estejam válidos, simular uma integração com a operadora de cartão de crédito que na verdade seja apenas um sorteio com 80% de chance de aprovar a compra e 20% de chance de reprovar.
    • Para boleto bancário, deve-se gerar um boleto em PDF com código de barras e os dados da compra. Não precisa ser um boleto pagável de verdade.
  • (Pagador) Consultar pagamentos realizados
  • (Pagador) Registrar dados do(s) cartão(ões) para facilitar compras futuras
  • (Recebedor) Consultar pagamentos recebidos
  • (Recebedor) Débito automático de mensalidade, descontando do crédito recebido de pagamentos (pois, para ser um recebedor nesse site, é preciso se cadastrar e pagar uma mensalidade)
  • (Recebedor) Aviso por e-mail de dois meses consecutivos de saldo negativo na conta


Sistema de controle de tráfego de ônibus

Tipos de usuários

  • Gerente
  • Passageiro
  • Motorista
  • Atendente de ponto de venda

Funcionalidades

  • (Motorista) Iniciar trajeto
  • (Motorista) Finalizar trajeto
  • (Passageiro) Liberar entrada ao passar cartão, validando se aquele ônibus está mesmo funcionando, debitando saldo do passageiro e verificando se ele possui saldo suficiente (considerar uma tolerância de até R$5 negativos)
  • (Atendente de ponto de venda) Recarregar cartão de passageiro
  • (Gerente) Consultar em tempo real os ônibus que estão circulando e a quantidade de passageiros que já entraram em cada um
